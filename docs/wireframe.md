# 画面設計・ワイヤーフレーム（Wireframe Overview）

最終更新: 2025-11-06  
Figma プロトタイプ: https://opt-mute-12091200.figma.site/

---

## 1. 画面マップ

```
Home ──┬─ Cocktails List (検索結果は同一画面内)
       ├─ Cocktail Detail (/cocktails/:id)
       ├─ Favorites (要ログイン)
       ├─ Sign Up
       ├─ Login
       └─ Email Confirmation 完了画面（トースト表示）
```

各ページは React Router で実装予定。ナビゲーションはヘッダー右上のボタンまたはサイドメニューから遷移する。

---

## 2. トップページ / 一覧ビュー

![Top Page](https://github.com/user-attachments/assets/65fdcb05-6be0-4a44-a582-90d8c7d0b7a6)

- ヒーローセクションには「本日の一杯」を配置。`GET /api/v1/cocktails/todays_pick` を叩き、日替わりのおすすめをカードで表示する。
- 右上にお気に入りページとログイン/ログアウトの操作がまとまったヘッダー。
- メインコンテンツは以下の 2 カラム:
  1. **フィルタパネル**（左）  
     - ベース酒（ラジオ＋複数選択対応）  
     - 検索キーワード（`q`）  
     - 材料テキスト入力（読点/カンマ区切り）  
     - 人気順/ID 順のトグル  
     - リセットボタンで `page=1` に戻す
  2. **カードグリッド**（右）  
     - 画像／名前（英日併記）／ベース／アルコール強度  
     - ページネーションコンポーネント（shadcn Pagination）
     - お気に入り状態を示すアイコン（ログイン済みのみ）

レスポンシブではフィルタパネルがドロワー内に折り畳まれ、カードは 1～2 列表示になる。

## 3. カクテル詳細ページ

![Cocktail Detail](https://github.com/user-attachments/assets/1d39cc7d-f6bc0-4837-9742-7b9e97346416)

### 上部ヒーロー
- 画像、英語名、日本語名を大きく表示。
- ベース・アルコール強度・技法（Build/Stir/Shake）をバッジ表示。
- お気に入りボタン（ログイン済みの場合はトグル可能）。
- 管理者はここから編集ボタンを表示し、モーダルで更新できるよう拡張予定。

### 詳細セクション
- **Ingredients**: 材料リストを表形式で表示（`position` 順）。英語/日本語表記と分量を併記。
- **How to Make**: 手順の英語/日本語テキスト。
- **Notes**: 説明文 (`description`) とグラス情報 (`glass`, `glass_ja`)。
- 関連カクテルのサジェストは将来対応とし、プレースホルダーを設置。

## 4. お気に入りページ

![Favorites](https://github.com/user-attachments/assets/072d9342-9c3f-42ea-82e8-71767e34606c)

- ログインユーザー専用。未ログインの場合はログインモーダルにリダイレクト。
- `GET /api/v1/favorites` を読み込み、作成日時降順でカード表示。
- カードにはカクテル基本情報＋「お気に入り解除」ボタンを配置。解除時はローカル状態と API の両方を更新。
- お気に入りがゼロの場合は空状態のイラストと CTA（検索へ戻る）。

## 5. 認証フロー

### 新規登録
![Sign Up](https://github.com/user-attachments/assets/f92d0aad-fa72-42fb-a28d-10fe8f4a44ee)

- 入力項目: 名前、メール、パスワード、パスワード確認。
- バリデーションエラーはフィールド下に表示し、Devise からの `errors` をマッピング。
- 完了後は「確認メールを送信しました」トーストを表示し、ログインページへ案内。

### ログイン
![Login](https://github.com/user-attachments/assets/4dec9f75-2747-46a6-88a3-9d13fdec0ba1)

- メール + パスワードでフォーム送信。
- 成功時は JWT を保存し、トップページへリダイレクト＆成功トースト。
- 失敗時は Devise のメッセージ（未確認/認証失敗）を表示。

### メール確認
- 確認リンク (`GET /confirmation`) を開くと「メールアドレスを確認しました」トーストを表示し、プロフィール情報を更新。
- トークン無効の場合はエラートースト＋再送フォームを表示。

## 6. フッター / グローバルコンポーネント

![Footer](https://github.com/user-attachments/assets/17b9f0f1-cbcd-4727-b07b-86ee1629a9b3)

- フッターにはサイト名、GitHub リポジトリリンク、利用規約、プライバシーポリシーのリンク。
- 全ページ共通のトースト通知とローディングスピナーを shadcn/ui で実装。
- 404 / 500 ページも簡易的なイラストで統一感を持たせる。

---

## 7. レスポンシブ設計

![Responsive](https://github.com/user-attachments/assets/41105c98-796d-4094-b1d4-d5952f133aae)

- モバイルでは検索フィルタをディスクロージャに格納し、カードは 1 列。
- ナビゲーションはハンバーガーメニューに変換し、ログイン状態はアイコンで表示。
- 主要コンポーネントは Tailwind のブレークポイント（`sm`, `md`, `lg`）で調整。
