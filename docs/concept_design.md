# コンセプトデザイン（Product Concept）

最終更新: 2025-11-06
目的: Today’s Cocktail の世界観・ユーザー価値を共有し、機能設計や UI/UX の意思決定をブレさせないための指針をまとめる。

---

## 1. プロダクトビジョン

> **家でもバーでも “今日の一杯” が見つかるパーソナル・カクテルガイド**

カクテル初心者が「名前だけでは味が分からない」「手持ちの材料で何が作れるか想像できない」という不安をなくし、写真と物語でカクテルを身近に感じられる体験を提供する。

---

## 2. ユーザーペルソナ

| ペルソナ   | 属性                      | ニーズ・課題                                                           |
| ---------- | ------------------------- | ---------------------------------------------------------------------- |
| Ayaka (28) | OL / 在宅ワークが多い     | 仕事終わりに自宅で軽く一杯。簡単・おしゃれ・写真映え重視。             |
| Ken (32)   | バー好きの会社員          | バーテンダーに頼りきり。メニューの味が想像できず注文に時間がかかる。   |
| Riku (21)  | 大学生 / サークル飲み担当 | 限られた予算で材料を揃えて皆を楽しませたい。作り方の段取りを知りたい。 |

---

## 3. ユーザーストーリー

1. **材料ベース探索**
   - Ayaka はストックしているジンとレモンから作れるレシピを探す。
   - ベース: gin、材料: lemon で検索 → 写真付きカードで候補を比較 → お気に入り保存。

2. **味の印象で選ぶ**
   - Ken は「さっぱり」「弱め」のカクテルを知りたい。
   - 度数やテイスティングノートをタグ化し、一目で判断できる UI を提供。

3. **作り方の段取り確認**
   - Riku は初めて作るカクテルを手際よく提供したい。
   - 手順、必要器具、難易度のヒントを分かりやすいステップで表示。

---

## 4. コア価値提案

- **Discover**: ベース酒・材料・味わいから自分好みのカクテルを探しやすい。
- **Learn**: 度数やテイスティングノート、由来などのバックグラウンドで理解を深める。
- **Save**: お気に入り登録で “My List” を構築し、次に作るレシピをすぐ呼び出せる。
- **Share (将来)**: SNS でのシェアやカクテルメモ機能を検討し、コミュニティ化を目指す。

---

## 5. デザイン原則

1. **Warm & Elegant**
   - バースタイルの落ち着いた配色（深いネイビー + ゴールドアクセント）。
   - 写真は高品質で、ガラスの透明感が伝わるライティング。

2. **Guided Discovery**
   - 初心者向けにフィルタとタグを構造化し、迷いを最小化。
   - レシピ詳細は “Step-by-Step” に分け、各ステップに小さなチップでアドバイス。

3. **Consistency & Accessibility**
   - 文字サイズとコントラストを WCAG AA 以上を意識。
   - 英日併記で材料名・説明を表示し検索性能を担保。

## 6. MVP 範囲

| 項目                      | 内容                              | コンセプトとの紐づき |
| ------------------------- | --------------------------------- | -------------------- |
| カクテル一覧・詳細        | 写真 / 味わい / 度数 / レシピ手順 | Discover, Learn      |
| ベース・材料検索          | 多角的な検索体験                  | Guided Discovery     |
| 今日の一杯                | 日替わり提案で習慣化              | Discover             |
| ユーザー登録 + お気に入り | 個人のレパートリー保存            | Save                 |
| メール認証付きログイン    | 体験のパーソナライズ基盤          | Consistency          |

---

## 7. ロードマップ（アイデア）

| フェーズ | 概要                                       | 期待価値                         |
| -------- | ------------------------------------------ | -------------------------------- |
| Phase 1  | MVP（検索 + レシピ + お気に入り）          | 個人で試す / ポートフォリオ公開  |
| Phase 2  | 味わいタグ・度数タグの追加、材料サジェスト | Discover 体験を強化              |
| Phase 3  | レコメンドロジック改善、SNS シェア         | コミュニティ拡張                 |
| Phase 4  | プロアカウント（バーテンダー登録）         | プロ視点のレシピ提供・収益化検討 |
